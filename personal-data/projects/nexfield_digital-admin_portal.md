## Description
Internal admin portal for a logistics client to manage vendors, orders, and user permissions. Solo frontend engagement with one backend developer and a PM, delivering a full role-based access control (RBAC) system from scratch.

## Achievements
- Designed and implemented a role-based access control (RBAC) system covering 4 user roles — Super Admin, Vendor, Operations, and Finance — with per-route and per-component permission guards, reducing client onboarding setup time by 2 days per new account.
- Delivered all 12 planned admin screens on schedule within a 9-month timeline as the sole frontend developer — managing feature scope, component architecture, and client feedback rounds independently.
- Built a reusable data table component with server-side pagination, sorting, and column configuration — reused across 8 of the 12 admin screens and cited by the client as a key usability improvement.

## Valued Inputs
- Initiated a component usage guide documenting each admin widget's props, variants, and usage constraints — used by the client's internal team when extending the portal post-handover.
- Proposed a permission matrix spreadsheet as a pre-implementation alignment artefact — resolved 6 role ambiguities with the client before any UI was built.
- Self-directed onboarding to VueJS and Ant Design at the start of the engagement — shipped first working screen within the first week of the project.
